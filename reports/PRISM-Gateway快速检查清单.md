# PRISM-Gateway 快速检查清单

> 7维度复盘 + Gateway行为准则的实用工具

---

## 📋 使用说明

本检查清单分为**两个部分**：
1. **Gateway实时检查**：任务开始前快速验证（<2分钟）
2. **Retrospective深度复盘**：任务完成后全面分析（10-30分钟）

选择适合的场景使用！

---

## Part 1: Gateway 实时检查（任务前）

### ⚡ 快速版（<2分钟）

> 适用场景：日常任务、常规操作

#### 维度1: 原则检查（MANDATORY）
- [ ] **搜索优先**：是否有足够的数据支撑？
  - 是 → 继续
  - 否 → 先进行信息搜索

- [ ] **验证真实**：信息来源是否可靠？
  - 已验证 → 继续
  - 未验证 → 交叉验证或标注"待确认"

- [ ] **不重复失败**：是否检查过失败模式库？
  - 已检查 → 继续
  - 未检查 → 查阅失败模式库

- [ ] **数据准确性**：关键数据是否验证过？
  - 已验证 → 继续
  - 未验证 → 标注假设并说明影响

- [ ] **用户期望**：是否清晰理解用户需求？
  - 已理解 → 继续
  - 不清晰 → 先确认需求

**结果**: □ 全部通过  □ 有警告（确认后继续）  □ 有违规（必须先修正）

---

#### 维度2+4: 模式与陷阱快速扫描
- [ ] **失败模式**：是否匹配到已知失败模式？
  - 无 → 继续
  - 有 → 查看避免策略

- [ ] **陷阱预警**：是否存在常见陷阱风险？
  - 无 → 继续
  - 有 → 查看避坑指南

**Gateway决策**: □ **通过**（执行任务）  □ **警告**（已知风险，确认后执行）  □ **阻断**（有MANDATORY违规，必须先修正）

---

## 💡 使用技巧

### 1. 灵活使用
- 小任务：仅使用Gateway快速检查
- 大任务：Gateway检查 + Retrospective完整复盘
- 失败案例：必须进行完整复盘

### 2. 持续改进
- 每次复盘后更新Gateway规则
- 每周回顾复盘质量
- 每月生成能力趋势报告

### 3. 团队协作
- 复盘报告共享给团队
- 提炼的知识加入团队库
- 定期组织复盘分享会
